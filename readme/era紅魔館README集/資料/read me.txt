/概要/
eratohoAをベースに何かのバリアントを作るため、ひとまずいろんなものを削ぎ落したバリアントです
csvはeratohoAから流用してますが適当な題材が見つかったら積み替える予定
ゲーム性やバランスは全く考慮しておりませんが、暇ならバグチェックにご協力ください。
EMUERA専用となっております

/主な変更点/
・SHOPでの購入処理の簡略化
	CHARA.CSV、ITEM.CSVの追加だけで店舗に並べることができるように
・いろいろな東方要素の削除
・労役、陥落素質方向性の削除
	淫乱、服従などはTALENTとして残っていますが、恋慕のみしかつきません
	また親愛はTALENT:3 = 2とするなど変数を圧縮しています
・仕様変更予定に抵触するコマンドをいくつか削除
	触手は全削除、あといくつか
	まだTEQUIPなどの変数としては残っています
・口上表示タイミングの変更
	SOURCE_CHECKの前に表示されていた口上をSOURCE_CHECKの最後尾に持って行きました
	口上表示のタイミングでUPや絶頂が確定しています
・口上システムの積み替え
	eratohoRevのシステムを参考に関数を細分化しています
	主人限定口上や口上セレクタにも対応
・衣装システムの改造
	衣装はEQUIPとして登録し、EQUIPの状況をTEQUIPに集約するという方法を取っています
	なので調教中に使うTEQUIPは
	TEQUIP:0 下半身着衣状況(ビット0=スカート 1=ずらし可 2=ずらし不可)
	TEQUIP:1 上半身着衣状況(0=なし 1=はだけ可 2=はだけ不可)
	だけです。
	ただし、EQUIPへの登録は未実装
・インサートシステム
	挿入状態をTEQUIPで管理してインサート状態で色々できるようにしました
・ビデオ仕様変更
	一回の撮影で同じ調教を繰り返す際にペナルティが与えられます
・コマンドフィルタ、調教メニュー登録の搭載
	文字列変数を使って変数をすっきりさせました
・調教画面における調教メニュー表示
	USERCOMから呼び出すことにより、いろいろ細工しやすくなるように
・射精周りの関数積み替え
	誰が誰にどのように、という情報はすべていくつかのTCVARに代入しています
	フラグを圧縮するとともに、追加、変更がしやすいように
・にんっしんっ
	射精周り関数に組み込んでいますが、具体的な処理は未実装です
・性別をビットに
	TALENT:2に女性器は1,男性器は2としてビットで管理するようにしています
・バストサイズの変数を圧縮
	TALENT:105 に絶壁から爆乳を-2～2として代入
・日常、夜這い、朝の一時、その他もろもろの追加要素の削除
	全部積み直す予定
