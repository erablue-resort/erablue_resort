・口上中に使えそうな条件や変数集積メモ

;〇NOWEX
;　今_絶頂してる、箇所。
;　　NOWEX:対象キャラ:Ｃ絶頂
;　　NOWEX:対象キャラ:Ｖ絶頂
;　　NOWEX:対象キャラ:Ａ絶頂
;　　NOWEX:対象キャラ:Ｂ絶頂
;　　NOWEX:対象キャラ:Ｓ絶頂
;　これらに、それぞれの絶頂強度が入っている
;　1:絶頂(10000超え)  2:強絶頂(20000超え)  3:超強絶頂(30000超え)  4:最強絶頂(40000超え)  5:究極絶頂(50000超え)
;　この強度の名前ダサいから変えたい……
;
;〇TCVAR
;　絶頂回数カウント。TCVAR.CSVも参照な。
;　　TCVAR:対象キャラ:うふふ中Ｃ絶頂累積
;　　TCVAR:対象キャラ:うふふ中Ｖ絶頂累積
;　　TCVAR:対象キャラ:うふふ中Ａ絶頂累積
;　　TCVAR:対象キャラ:うふふ中Ｂ絶頂累積
;　　TCVAR:対象キャラ:うふふ中Ｓ絶頂累積
;　これらはうふふ終了でリセットされる、絶頂回数を累積カウントする変数
;　当然非うふふ中のセクハラなどでは変動しないので注意
;　/
;　SIF TCVAR:対象キャラ:うふふ中Ｓ絶頂累積 >= 30
;　　このうふふ中にS絶頂30回以上
;　/
;　……といった形で利用可能。

;
;〇MODE（シチュエーション、状況）
;　「セックス中であるか」などの確認用
;　通常（非うふふ）、うふふ、飲み会、マッサージの四種かな？　がある
;　/
;　SIF CFLAG:PLAYER:うふふ
;　　プレイヤー（多くの場合はオーナーだね）がうふふ中
;　SIF CFLAG:TARGET:うふふ
;　　ターゲットがうふふ中
;　SIF CFLAG:PLAYER:飲み会
;　　プレイヤー（多くの場合はオーナーだね）が飲み会参加中
;　SIF CFLAG:TARGET:飲み会
;　　ターゲットが飲み会参加中
;　SIF SAVESTR:ゲームフェイズ管理 == "マッサージモード"
;　　オーナーと対象キャラがマッサージ中
;　/
;　通常に関しては、上記MODEでないことで判定するしかない？
;
;〇体位MODE（セックスしている、フェラしている、など）
;　今なにをしているか、の確認用。
;　下記のごとき判定文で検索が可能。モードがあるなら1が、無いなら0が返ってくる。
;　/
;　「MODE_存在判定_プレイヤー側(判定モード名, 判定キャラ番号)」
;　「MODE_存在判定_ターゲット側(判定モード名, 判定キャラ番号)」
;　「MODE_存在判定_いずれか側(判定モード名, 判定キャラ番号)」
;　「MODE_存在判定_完全一致(判定モード名, 実行キャラ番号, 対象キャラ番号)」
;　/
;　SIF MODE_存在判定_プレイヤー側("全裸散歩", PLAYER)
;　　プレイヤーが、全裸散歩をさせている
;　SIF MODE_存在判定_ターゲット側("ハーヴィンオナホＶ", TARGET)
;　　ターゲットが、ハーヴィンオナホＶをされている
;　SIF MODE_存在判定_いずれか側("キス系", PLAYER)
;　　プレイヤーが、キスをしているか、されている
;　SIF MODE_存在判定_完全一致("Ａ挿入系", PLAYER, TARGET)
;　　プレイヤーが、ターゲットに、A挿入系コマンド（A正常位、A後背位など、体位は問わない）を実行している
;　SIF MODE_存在判定_完全一致("正常位", PLAYER, TARGET)
;　　プレイヤーが、ターゲットに、正常位を実行している（後背位などではダメ）
;　/
;　「判定モード名」に関しては、個別のコマンド名（"寝バック"、"Ａ騎乗位"など）のほか、カテゴリ指定でも検索できる。
;　　ALL
;　　ＶＡ挿入系
;　　キス系
;　　パイズリ系
;　　フェラチオ系
;　　口戯系
;　　シックスナイン可能
;　　手コキ系
;　　素股系
;　　乳首吸い系
;　　アイテム系
;　　ハーヴィンオナホ系
;　　自慰系
;　　Ｃ自慰系
;　　Ｖ自慰系
;　　Ａ自慰系
;　　Ｂ自慰系
;　　触手系
;　　Ａ挿入系
;　　Ｖ挿入系
;
;〇コマンド履歴:0～4、派生コマンド履歴:0～4
;　今まで実行したコマンド番号の履歴。
;　基本的に「コマンド履歴」の方に入るが、コマンド履歴が-1の場合のみ派生コマンド履歴の方に入る
;　分けているのはコマンド履歴が数値、派生コマンド履歴が文字列のため
;
;〇コマンド履歴:5～9
;　今まで実行したコマンドのTARGETを格納。コマンド履歴:0～4に対応している。
;
;〇コマンド履歴:10～14
;　今まで実行したコマンドの使用した位置を格納。コマンド履歴:0～4に対応している。
;
;〇コマンド履歴:15～19
;　今まで実行したコマンドの使用したマップ種別を格納。コマンド履歴:0～4に対応している。
;
;〇SELECTCOM、RSTR:SELECTCOM_S受け渡し
;　今この瞬間使用したコマンドの番号
;　SELECTCOMが数字、RSTR:SELECTCOM_S受け渡しが文字列で、基本的にどちらかだけ入る
;　「IF RSTR:SELECTCOM_S受け渡し != ""」でRSTR:SELECTCOM_S受け渡しが空っぽなのをでチェックしてから
;　SELECTCOMの中身を見るのが事故がなくていいかも
;
;〇TEQUIP
;　そこをなにかに使ってる場合。
;　「口をなにかで塞がれていて喋れない」などの場合に使えるか。
;　　TEQUIP:PLAYER:ペニス
;　　TEQUIP:PLAYER:口
;　……など。TEQUIP.csv参照な。
;
